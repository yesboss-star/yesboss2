SCREEN 6 — DYNAMIC AI FILE REQUEST SCREEN
------------------------------------------
FRONTEND:
- Industry-specific file suggestions from hardcoded INDUSTRY_FILE_SUGGESTIONS (60+ industries) + MICRO_VERTICAL_FILE_SUGGESTIONS (40+ micro-verticals)
- File upload: PDF, Excel, Word, Images (max 25MB, max 10 files batch)

BACKEND (core/file_processor.py):
- ALLOWED_EXTENSIONS: .pdf, .xlsx, .xls, .png, .jpg, .jpeg, .docx, .txt, .csv
- Text extraction: PyMuPDF (fitz) + PyPDF2 fallback, pandas (Excel), Pillow + pytesseract (images), python-docx (Word)
- Chunking: 1000 chars, 200 overlap
- Embeddings: text-embedding-3-small via xAI/OpenAI
- Storage: MongoDB (documents collection) + Qdrant (documents collection)
- Search: POST /api/v1/files/search

NO PaddleOCR — uses pytesseract for OCR.

SCREEN 8 — PERSONA + ORGANIZATIONAL INTELLIGENCE CHAT
------------------------------------------------------
FRONTEND (Owner Onboarding):
- Persona questions generated by AI via POST /api/v1/chatbot/persona/generate-question
- Question context: org_name, industry, micro_vertical, company_size, domain, social_links, previous_answers
- 3 options per question + custom text input
- After each answer → AI generates next follow-up question
- After certain point: "Do you have more time?" prompt (persona-more-time step)
- Fallback questions (5 hardcoded) if AI fails

BACKEND (api/chatbot.py):
- generate_persona_question: builds context from org data, previous answers → AI generates next question
- Uses get_ai_response with temperature=0.8
- Returns {question, options, time_estimate, need_more_time, question_number}

Backend OnboardingChatbot (core/chatbot.py):
- start_conversation: begins with welcome + first question
- process_message: handles conversation loop
- 8 fixed question topics: company_description, industry, company_size, goals, challenges, workflows, decision_making, growth
- Conversation stored in MongoDB (conversations) + Qdrant (conversations)

API: POST /api/v1/chatbot/start, POST /api/v1/chatbot/message, GET /api/v1/chatbot/conversation/{id}, GET /api/v1/chatbot/user/{user_id}

SCREEN 14 — CONTINUOUS LEARNING ENGINE (Background)
----------------------------------------------------

- ContinuousLearning class with MongoDB storage
- Collections: workflows, task_outcomes, bottlenecks, learning_patterns

NO LoRA Fine-Tuning, NO PyTorch, NO HuggingFace, NO PEFT, NO Unsloth in code.
